1. INTRODUCTION GIT

Git adalah version control system paling populer. dibuat oleh Linus Torlvads pada tahun 2005. Dan telah dikelola oleh Junio Hamono sejak saat itu

GIT mencatat setiap perubahan file di code atau sourcode kita

Manfaat GIT

  1. Bisa menyimpan seluruh versi source code
  2. Berkolaborasi membangun sistem menjadi mudah
  3. Bisa berkontribusi di proyek source code
  4. Memudahkan dalam tracking perubahan
  5. Memahami dalam deploy modern

2. FUNDAMENTAL GIT

Berisikan perintah-perintah yang digunakan untuk mengoperasikan GIT

Beberapa perintah yang ada pada git :

Git --version

Melihat version git yang digunakan

Git init

  • Digunakan untuk menginisialisasi repository
  • Memberi tahu folder yang dijadikan repository
  • Kita tidak dapat menggunakan git, tanpa menginisialisasinya

Git config

  • Konfigurasi username dan useremail
  • Biasanya disamakan dengan akun gitlab / github
  • config global untuk konfigurasi ke server
  • Config local untuk konfigurasi local

Git config --list

Untuk melihat semua konfigurasi yang ada di git

Git status

  • Digunakan untuk mengetahui status file
  • Status file secara umum ada 3. Untracked, staging area, modified

Git add .

  • Untuk memasukkan file ke staging area
  • Penulisan : git add (nama_file) atau git add .

Git commit

  • Untuk mencatat perubahan yang dilakukan
  • Cara Penulisan : git commit -m "(comment / nama_perubahan)"

Git log

  • Untuk melihat riwayat commit, beserta comment yang pernah dilakukan
  • Tekan q untuk keluar dari log

Git clone

  • Digunakan untuk mengcopy repository dari server ke local
  • Cara Penulisan : git cloen "URL"

Git Push

  • Digunakan untuk mengirim commit ke server
  • Cara Penulisn : git push orign "nama_branch"

Git Remote

  • Semacam api untuk berinteraksi dari client dengan server
  • Cara Penulisn : git remote add origin "URL"

Git Pull

  • Untuk mengambil commit dari server jika ada perubahan diserver yang dibuat oleh branch lain
  • Cara Penulisn : git pull orign "nama_branch"
